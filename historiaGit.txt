O núcleo do Linux, um projeto de código aberto, inicialmente usava correções e arquivos para mudanças de código até 2002, quando passou a usar o sistema de controle de versão BitKeeper. Em 2005, devido a desentendimentos com a empresa responsável pelo BitKeeper e ao fato de a ferramenta ter se tornado paga, a comunidade Linux, liderada por Linus Torvalds, decidiu criar uma nova ferramenta de controle de versão. O Git foi desenvolvido com objetivos de alta velocidade, simplicidade, suporte robusto para desenvolvimento não-linear com muitos ramos, e eficiência em projetos grandes. Desde seu lançamento, o Git evoluiu para ser rápido, eficiente e manter suas qualidades iniciais.
